# Custom (Modified Genetic) Algorithm for 0/1 Knapsack

## About the Algorithm

The Genetic Algorithm (GA) is a heuristic inspired by Charles Darwin's theory of natural evolution. This algorithm reflects the process of natural selection where the fittest individuals are selected for reproduction in order to produce offspring of the next generation.

As a heuristic, the GA does not guarantee the optimal solution. However, it is very effective at finding high-quality, near-optimal solutions in a fraction of the time required by exact algorithms, especially for large problem sizes.

## Implementation

### Modifications

In this version of the GA, we introduce several modifications aimed at improving the quality of solutions and the rate of convergence towards them. We do this by implementing intelligent heuristics in place of pure randomness, such as in individual repair, population initialisation, and generational evolution.

#### Greedy Repair Strategy

The base GA repairs overweight individuals by randomly removing items till the knapsack is valid. This version instead uses a **two-phase repair approach**:

**Phase 1: Removal** — Iteratively removes items with the lowest value/weight ratio until the individual is within capacity. This ensures we remove the "least valuable" items first, minimizing the loss in solution quality.

**Phase 2: Greedy Filling** — Once the individual is feasible, greedily attempts to add items with the highest value/weight ratio that fit within the remaining capacity. This maximises the utilisation of the knapsack and guides the population towards higher-quality solutions. **Note:** To balance exploration and exploitation, greedy filling is only applied every 5 generations rather than every generation.

#### Greedy Population Seeding

Instead of starting with a purely random population, the initial population includes a small percentage of individuals generated using a greedy approach:

- **Greedy Individuals** (~5% of population) — Generated by greedily selecting items in order of decreasing value/weight ratio until capacity is exhausted. These high-quality seed solutions provide a strong starting point.
- **Random Individuals** (~95% of population) — Randomly initialised to maintain population diversity and enable exploration of the solution space.

All individuals in the initial population are then repaired using the greedy repair strategy, ensuring the population starts in a feasible state while retaining the benefits of both approaches.

#### Elitism

To ensure that the best solution found so far is not lost in subsequent generations, this algorithm incorporates elitism. In each generation, the best individual (the one with the highest fitness) from the current population is automatically carried over to the next generation. This guarantees that the quality of the solution in the population can only increase or stay the same over time, leading to better and more consistent results.

#### Heuristic Crossover

The base GA uses uniform crossover where each bit is randomly inherited from either parent. This version implements a **heuristic crossover** strategy:

- **Agreement bits** — When both parents have the same bit value, both children inherit that value.
- **Disagreement bits** — When parents differ, child1 initially receives the fitter parent's bit, child2 receives the other's. A 50% random swap is then applied to create actual genetic mixing rather than parent cloning.

This approach preserves consensus genetic material while still allowing exploration at points of disagreement, leading to more effective recombination.

#### Diversity-Aware Mutation

Instead of pure random mutation, this version implements **diversity-preserving mutation**:

- **Item frequency tracking** — Tracks how often each item appears across the top individuals in the population.
- **Mutation bias** — Protects rare items (low frequency) from being removed and avoids adding already-common items (high frequency).
- **Geometric skip distribution** — Uses a geometric distribution to skip between mutations, improving efficiency over per-bit probability checks.

This prevents the population from converging prematurely on a single solution and maintains genetic diversity throughout evolution.

#### Adaptive Mutation Rate

The mutation rate automatically adjusts based on population convergence:

- **Stagnation detection** — Monitors best fitness over a sliding window (default: 10 generations).
- **Rate increase** — When fitness stagnates (no improvement), mutation rate increases by 1.5× (capped at 0.1) to escape local optima.
- **Rate reset** — Returns to base rate when improvement resumes.

#### Stagnation Recovery

When the population becomes stagnant:

- **Worst individual reinitialisation** — The bottom 10% of the population (by fitness) is replaced.
- **Median-based recovery** — Replaced individuals are cloned from the median-fitness individual, then undergo aggressive double mutation.
- **Repair and continue** — Reinitialised individuals are repaired to maintain feasibility.

This mechanism helps escape local optima by injecting controlled diversity without destroying the best solutions.

#### Per-Generation Rank Precomputation

Instead of finding the best individual via linear search each time, population ranks are **precomputed once per generation**:

- Population indices are sorted by fitness in descending order (O(P log P)).
- Elitism, diversity tracking, and stagnation recovery all reuse these precomputed ranks.

This amortises the cost of multiple fitness-based lookups over a single sort operation.

### Complexity

#### Individual Operations

| Function | Time Complexity | Space Complexity | Notes |
|----------|-----------------|------------------|-------|
| `Individual::calculateMetrics()` | O(N) | O(1) | Iterates through all N items once |
| `Individual::getFitness()` | O(1) | O(1) | Amortised constant time with caching |
| `Individual::repair()` | O(N) | O(1) | Two-phase removal and greedy filling; worst case removes/adds all items |
| `Individual::mutate()` | O(N/M) expected | O(1) | Geometric skip distribution; M = mutation rate; expected ~M×N mutations |

#### Preprocessing

| Function | Time Complexity | Space Complexity | Notes |
|----------|-----------------|------------------|-------|
| `SortedItemsData::preSortItems()` | O(N log N) | O(N) | Computes value/weight ratios O(N), sorts N items O(N log N) |

#### Per-Generation Operations

| Function | Time Complexity | Space Complexity | Notes |
|----------|-----------------|------------------|-------|
| `SelectionData::precomputeRanks()` | O(P log P) | O(P) | Sorts population indices by fitness once per generation |
| `DiversityData::updateFrequency()` | O(S × N) | O(N) | S = sample size (10); samples top individuals for frequency tracking |
| `ConvergenceData::update()` | O(1) | O(W) | W = stagnation window (10); sliding window check |
| `reinitialiseWorstIndividuals()` | O(P log P + R × N) | O(P) | R = reinit count (P/10); only runs during stagnation |

#### Population Initialisation

| Function | Time Complexity | Space Complexity | Notes |
|----------|-----------------|------------------|-------|
| `generateGreedyIndividual()` | O(N) | O(1) | Iterates through pre-sorted items; greedy selection |
| `generateRandomIndividual()` | O(N) | O(1) | Batched random bits (32 at a time) for efficiency |
| `generateInitialPopulation()` | O(P × N) | O(P × N) | Creates P individuals with ~5% greedy-seeded, repairs all |

#### Genetic Operators

| Function | Time Complexity | Space Complexity | Notes |
|----------|-----------------|------------------|-------|
| `selection()` | O(1) | O(1) | Tournament selection with 4 random individuals, fitness cached |
| `crossover()` | O(N) | O(1) | Heuristic crossover with incremental metric computation (no recalculation) |
| `nextGeneration()` | O(P log P + P × N) | O(P) | Includes rank precomputation, diversity update, and P offspring creation |

#### Overall Algorithm

**`solveKnapsackGenetic()`**

- **Time Complexity**: **O(G × (P log P + P × N) + N log N) ≈ O(G × P × N)**
  - Pre-sort items: O(N log N)
  - Initialise convergence tracking: O(1)
  - Initial population generation: O(P × N)
  - G generations of evolution: G × O(P log P + P × N)
    - Rank precomputation: O(P log P)
    - Diversity frequency update: O(S × N) where S ≪ P
    - Convergence check: O(1)
    - Offspring creation: O(P × N)
    - Stagnation recovery (when triggered): O(P log P + R × N)
  - Finding best individual: O(P)
  - Collecting selected items: O(N)
  - Total: dominated by O(G × P × N)

- **Space Complexity**: **O(P × N)**
  - Two populations: 2 × P individuals × N bits each
  - `SORTED_DATA.items` array: O(N)
  - `SELECTION_DATA.populationRanks`: O(P)
  - `DIVERSITY_DATA.itemFrequency`: O(N)
  - `CONVERGENCE_DATA.fitnessHistory`: O(W) where W = 10
  - Item arrays (`weights`, `values`): O(N)
  - Result `selectedItems`: O(N) worst case
  - Total: dominated by O(P × N)

Where:

- **N** = number of items
- **P** = `populationSize` (default: 60)
- **G** = `maxGenerations` (default: 30)
- **S** = sample size for diversity tracking (10)
- **W** = stagnation window size (10)
- **R** = reinitialisation count during stagnation (P/10)

#### Fixed Hyperparameters

Unlike the previous version with dynamic scaling, this implementation uses **fixed default hyperparameters** (P=60, G=30) optimised for general performance across problem sizes. This simplifies tuning while still allowing command-line overrides for specific use cases.

| Parameter | Default Value | Override Flag |
|-----------|---------------|---------------|
| Population Size | 60 | `--population_size` |
| Max Generations | 30 | `--max_generations` |
| Crossover Rate | 0.53 | `--crossover_rate` |
| Mutation Rate | 0.013 | `--mutation_rate` |
| Reproduction Rate | 0.15 | `--reproduction_rate` |
| Random Seed | Random | `--seed` |

With P=60 and G=30, the effective complexity is **O(1,800N + N log N) ≈ O(N)** for practical purposes, making the algorithm efficient for large-scale instances.
